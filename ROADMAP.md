**ავტორი:** ნეო (AI Development Manager)
**თარიღი:** 2026-03-09
**მიზანი:** პასუხების ხარისხის რადიკალური გაუმჯობესება და ძიების სიღრმის გაფართოება

## ფაზა 1: ძიების სიღრმე (2 კვირა)

### 1.1 — მრავალწყაროიანი ძიება
ახლა: Perplexity + 5 API (ClinicalTrials.gov, PubMed, Europe PMC, EU CTR, WHO ICTRP)

**დასამატებელი წყაროები:**
- **OpenAlex** — 250M+ აკადემიური ნაშრომი, უფასო API, PubMed-ზე ფართო
- **Cochrane Library API** — სისტემატური მიმოხილვები (evidence-based medicine-ის ოქროს სტანდარტი)
- **DrugBank API** — მედიკამენტების მონაცემთა ბაზა (ინტერაქციები, დოზირება, ალტერნატივები)
- **OrphaNet API** — იშვიათი დაავადებები (საქართველოსთვის კრიტიკული — ბევრ იშვიათ დაავადებაზე ინფო არ არსებობს ქართულად)
- **KEGG Disease** — დაავადებების მოლეკულური მექანიზმები
- **FDA API** — წამლების ახალი approval-ები, safety alerts
- **Google Scholar Scraper** — ციტირებების რაოდენობა, impact factor

### 1.2 — ქართული სამედიცინო ლექსიკონის გაფართოება
ახლა: 50 ტერმინი Vercel-ზე, ~60 FastAPI-ზე

**გეგმა:**
- 500+ ტერმინიანი ქართულ-ინგლისური სამედიცინო ლექსიკონი
- წყარო: თბილისის სამედიცინო უნივერსიტეტის ტერმინოლოგია, WHO ICD-11 ქართული თარგმანი
- ავტომატური ტერმინების ამოცნობა ტექსტიდან (NER — Named Entity Recognition)
- სინონიმების მხარდაჭერა (მაგ: "შაქრიანი დიაბეტი" = "დიაბეტი" = "diabetes mellitus")

### 1.3 — კონტექსტური ძიება
ახლა: ერთი query → ერთი შედეგი

**გეგმა:**
- **Multi-hop ძიება** — პირველი შედეგებიდან ახალი query-ების გენერაცია
  - მაგ: "HIE მკურნალობა" → პოულობს "hypothermia therapy" → ეძებს "hypothermia therapy outcomes neonatal"
- **Related conditions** — მსგავსი/თანმხლები დაავადებების ავტომატური ძიება
- **Treatment pathway mapping** — მკურნალობის ეტაპების ჯაჭვის აგება

## ფაზა 2: პასუხების ხარისხი (2 კვირა)

### 2.1 — Evidence grading სისტემა
ახლა: ყველა შედეგი თანაბარია

**გეგმა:**
- ყველა შედეგს Evidence Level (I-V):
  - I: სისტემატური მიმოხილვა / მეტა-ანალიზი
  - II: რანდომიზებული კონტროლირებული კვლევა (RCT)
  - III: კოჰორტული კვლევა
  - IV: case study / case series
  - V: ექსპერტის აზრი
- ვიზუალური მარკერი (🟢🟡🔴) evidence სიძლიერის მიხედვით
- ფილტრაცია evidence level-ით

### 2.2 — Fact verification pipeline
ახლა: Claude ენდობა Perplexity-ს output-ს

**გეგმა:**
- **Cross-reference check** — ყველა ფაქტი მინიმუმ 2 წყაროდან
- **Recency check** — 5 წელზე ძველი კვლევების მონიშვნა
- **Retraction check** — გამოქვეყნებული მაგრამ უკან გაწვეული კვლევების ამოცნობა (Retraction Watch API)
- **Conflict of interest** — ფარმაცევტულ კომპანიებთან კავშირის მონიშვნა

### 2.3 — პასუხის სტრუქტურის გაუმჯობესება
ახლა: სათაური + წყარო + ტექსტი + ტეგები

**გეგმა:**
- **Executive Summary** — 3 წინადადებიანი შეჯამება ზემოთ
- **Visual timeline** — კვლევების ქრონოლოგიური ვიზუალიზაცია
- **Comparison table** — მკურნალობის ვარიანტების შედარება (ეფექტურობა, გვერდითი მოვლენები, ღირებულება)
- **საქართველოსთვის რელევანტურობა** — ხელმისაწვდომია თუ არა ეს მკურნალობა საქართველოში/რეგიონში
- **მოქმედების ნაბიჯები** — კონკრეტული "რა გააკეთოს პაციენტმა" სექცია

### 2.4 — პერსონალიზაცია
ახლა: ყველა იღებს ერთნაირ პასუხს

**გეგმა:**
- ძიების ისტორიის გათვალისწინება
- პაციენტის პროფილი (ასაკი, სქესი, თანმხლები დაავადებები)
- შედეგების ადაპტაცია — სამედიცინო პროფესიონალი vs პაციენტი (სიღრმის დონე)

## ფაზა 3: კლინიკების ძიების რევოლუცია (2 კვირა)

### 3.1 — რეალური მონაცემთა ბაზა
ახლა: Claude/Perplexity-ზე დაფუძნებული (ხშირად არაზუსტი ფასები)

**გეგმა:**
- **საქართველოს კლინიკების ბაზა** — ყველა მთავარი კლინიკა ფასებით, სპეციალობებით
- **თურქეთი** — Medical tourism-ის ტოპ კლინიკები (ქართველთა #1 მიმართულება)
- **ისრაელი** — Hadassah, Sheba, Ichilov
- **გერმანია** — Charité, Heidelberg
- **ინდოეთი** — Apollo, Fortis (ბიუჯეტური ვარიანტი)
- ფასების რეგულარული განახლება (scraping + manual verification)

### 3.2 — ფასების შედარება
- მკურნალობის ღირებულება ქვეყნების მიხედვით
- მგზავრობის + საცხოვრებლის ხარჯების ჩათვლით
- სადაზღვევო დაფარვის ინფორმაცია

### 3.3 — ექიმების რეიტინგი
- სპეციალისტების პროფილები გამოცდილებით
- პაციენტთა რეკომენდაციები
- პუბლიკაციების რაოდენობა (h-index)

## ფაზა 4: AI გაძლიერება (3 კვირა)

### 4.1 — Multi-agent არქიტექტურა
ახლა: ერთი Claude call

**გეგმა:**
- **Agent 1: Researcher** — წყაროების მოძიება და ფილტრაცია
- **Agent 2: Analyst** — შედეგების ანალიზი და evidence grading
- **Agent 3: Translator** — სამედიცინო ტერმინოლოგიის ზუსტი თარგმანი
- **Agent 4: Advisor** — პაციენტისთვის გასაგები რეკომენდაციების ფორმულირება
- **Agent 5: Fact-checker** — ჯვარედინი შემოწმება

### 4.2 — RAG (Retrieval-Augmented Generation)
- ქართული სამედიცინო ტექსტების ბაზა (თსსუ სახელმძღვანელოები, სამედიცინო ჟურნალები)
- Vector DB (Pinecone/Weaviate) სემანტიკური ძიებისთვის
- ქართული embedding მოდელი სამედიცინო ტერმინებისთვის

### 4.3 — სამედიცინო ჩატბოტის გაძლიერება
ახლა: rule-based keyword matching

**გეგმა:**
- Claude-ზე დაფუძნებული ინტელექტუალური ჩატბოტი
- კონტექსტის შენარჩუნება (მრავალტურიანი საუბარი)
- ძიების შედეგებზე follow-up კითხვები
- "ამიხსენი უფრო მარტივად" ფუნქცია

## ფაზა 5: პლატფორმა (4 კვირა)

### 5.1 — მომხმარებლის ანგარიში
- რეგისტრაცია/ავტორიზაცია (Supabase — უკვე backend მზადაა)
- ძიების ისტორია
- შენახული კვლევები (bookmarks)
- პერსონალური dashboard

### 5.2 — შეტყობინებები
- ახალი კვლევების alert — "შეგატყობინე როცა HIE-ზე ახალი კვლევა გამოქვეყნდება"
- კლინიკური კვლევების მონიტორინგი — ახალი trial-ების ძიება პაციენტის პროფილის მიხედვით
- ფასების ცვლილებების alert

### 5.3 — ექიმის პანელი
- ექიმებისთვის სპეციალური ინტერფეისი
- პაციენტთან შედეგების გაზიარება
- რეფერალის სისტემა (ექიმი → კლინიკა)

### 5.4 — API as a Service
- RESTful API სხვა ქართული სამედიცინო პლატფორმებისთვის
- Widget — ჩასმა სხვა საიტებზე
- Telegram/WhatsApp ბოტი
